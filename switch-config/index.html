SC1b

Per-Rack Leaf Switches

Disaggregated deployments use one leaf pair per rack (2 ToR switches each) joined to a shared VXLAN-EVPN spine fabric. Each rack gets its own BGP ASN, loopback pair, Mgmt SVI IPs, underlay P2P links to the spine, and intra-pair iBGP peering — all editable below. Up to 8 racks (16 ToR switches) are supported.

Common VLAN IDs, QoS, TACACS, NTP, and the shared Border Router ASN are configured in SC2/SC3 below and applied to every rack.

QA

Validate Existing ToR Switch Configuration

Paste the output of show running-config (Cisco NX-OS) or show running-configuration (Dell OS10) from a ToR switch to validate that QoS settings meet Azure Local requirements for storage (RDMA) and cluster heartbeat traffic.

✅ This audit checks: PFC (IEEE 802.1Qbb) on CoS 3 for lossless storage, ETS (IEEE 802.1Qaz) bandwidth reservations (Storage ≥50%, Cluster 1–2%), ECN on the storage queue, MTU 9216 (jumbo frames), system QoS policy application, VLAN trunking on host-facing interfaces, and interface-level PFC enablement. Which checks are required vs. optional depends on the Deployment Type selected below.
🔒 Privacy: All processing is performed entirely in your browser. No switch configuration data is stored, transmitted, or sent to any external service. Sanitize any passwords or shared secrets before pasting.
